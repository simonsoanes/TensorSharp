快速开始

从一台空机器到流式回复。原生 GGML 库会在首次构建时自动编译，因此唯一需要手动安装的是 .NET SDK（如需加速，再加上 GPU 工具链）。

1 · 环境要求

  • .NET 10 SDK —— 必需。
  • git + 网络访问 —— 首次构建时，原生构建会把 ggml 源码从 github.com/ggml-org/ggml 克隆到 ExternalProjects/ggml/。设置 TENSORSHARP_GGML_NO_UPDATE=1 可在离线重建时跳过网络更新。
  • 一个 GGUF 模型文件 —— 例如来自 Hugging Face。见模型下载。

各平台工具链（仅 GPU 加速需要）

平台用于安装
macOS（Metal）ggml_metal / mlxCMake 3.20+ 与 Xcode 命令行工具。MLX 还会额外构建 libmlxc。
Windowsggml_cuda / cudaCMake 3.20+、Visual Studio 2022 C++ 构建工具、NVIDIA 驱动 + CUDA Toolkit 12.x（含 cuBLAS）。
Linuxggml_cuda / cudaCMake 3.20+、NVIDIA 驱动 + CUDA Toolkit 12.x（含 cuBLAS）。
Windows / Linux（任何 Vulkan GPU）ggml_vulkan当主机存在 Vulkan 运行时（loader 已安装）时自动启用；用 --no-vulkan（或 TENSORSHARP_GGML_NATIVE_ENABLE_VULKAN=OFF）退出。Windows 上未安装 SDK 时会自动准备便携 Vulkan 工具链；Linux 上请安装如 libvulkan-dev glslc spirv-headers。运行时需要 Vulkan 1.3 驱动。
任意（仅 CPU）cpu / ggml_cpu除 .NET SDK 外无需任何东西。

2 · 构建

克隆并构建整个解决方案。首次构建还会编译原生 GGML 桥接（可能需要一段时间，尤其是 CUDA —— 后续构建很快）。

git clone https://github.com/zhongkaifu/TensorSharp.git
cd TensorSharp
dotnet build TensorSharp.slnx -c Release

或只构建其中一个应用：

# 控制台应用
dotnet build TensorSharp.Cli/TensorSharp.Cli.csproj -c Release

# Web 应用
dotnet build TensorSharp.Server/TensorSharp.Server.csproj -c Release

CLI 二进制位于 TensorSharp.Cli/bin/...，服务器位于 TensorSharp.Server/bin/...。如需启用 CUDA 或 Vulkan 的原生构建、手动原生构建或 MLX，见构建原生库。

📦

不想从源码构建？Release Binaries 工作流会发布自包含归档（Windows/Linux CPU 与 CUDA、macOS arm64），其中已捆绑 .NET 10 运行时与原生库 —— 无需另行安装即可运行。见后端。

3 · 下载模型

TensorSharp 加载 GGUF 格式的模型。一个轻量、经过充分测试的起点是 ggml-org/gemma-4-E4B-it-GGUF 中的 Gemma-4-E4B (Q8_0)。选择适合你硬件的量化 —— 低内存用 Q4_K_M，更高质量用 Q8_0。完整列表见模型页。

🧩

对于多模态模型，还需下载匹配的投影器（mmproj）文件。如果把它放在模型旁并使用可识别的名称（如 gemma-4-mmproj-F16.gguf），会被自动检测。

4 · 首次运行

根据你的硬件选择 --backend。每个后端都产生正确输出；它们只在速度上不同。

方式 A —— 一次性生成（CLI）

echo "用一句话解释专家混合（MoE）。" > prompt.txt

# macOS（Apple Silicon）
./TensorSharp.Cli --model gemma-4-E4B-it-Q8_0.gguf --input prompt.txt --backend ggml_metal

# Windows / Linux + NVIDIA
./TensorSharp.Cli --model gemma-4-E4B-it-Q8_0.gguf --input prompt.txt --backend ggml_cuda

# Windows / Linux + AMD / Intel / NVIDIA（GGML Vulkan；存在 Vulkan 运行时时自动构建启用）
./TensorSharp.Cli --model gemma-4-E4B-it-Q8_0.gguf --input prompt.txt --backend ggml_vulkan

# 可移植 / 调试（无 GPU）
./TensorSharp.Cli --model gemma-4-E4B-it-Q8_0.gguf --input prompt.txt --backend cpu

方式 B —— 交互式聊天（REPL）

./TensorSharp.Cli --model gemma-4-E4B-it-Q8_0.gguf -i --backend ggml_metal

逐轮输入消息；用 斜杠命令（如 /reset、/think on 或 /image photo.png）控制会话。

方式 C —— 浏览器 UI + HTTP API（服务器）

./TensorSharp.Server --model gemma-4-E4B-it-Q8_0.gguf --backend ggml_metal
# 打开 http://localhost:5000

它在同一端口上同时提供聊天 UI 与兼容 Ollama 和 OpenAI 的端点。
